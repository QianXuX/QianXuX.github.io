Machine Learning Introduction

Posted on 2017-02-13   |   In 学习笔记 , Coursera Machine Learning   |  

Introduction

What is Machine learning?

Machine learning definition

Given experience E, task T, measurement P; We define that the machine has learnt from E if P in T has increased after feeding E.

E = the experience of playing many games of checkers

T = the task of playing checkers.

P = the probability that the
program will win the next game.

Supervised Learning

Difference between Regression and Classification. Regression is used to predict continuous values. Classification is used to predict which class a data point is part of discrete value.

Regression

Example: Given the size of an apartment, predict the price one can sell

Classification

Example: Given the color of some fruits, predict what type of fruit it is

Unsupervised Learning

Given the data, we don’t know the corresponding labels in advance. We can derive structure from data where we don’t necessarily know the effect of the variables.

Clustering

Example: Take a collection of 1,000,000 different genes, and find a way to automatically group these genes into groups that are somehow similar or related by different variables, such as lifespan, location, roles, and so on

Non-Clusting

Example: The “Cocktail Party Algorithm”, allows you to find structure in a chaotic environment. (i.e. identifying individual voices and music from a mesh of sounds at a cocktail party).

Effective Java notes - Methods

Posted on 2017-02-09   |   In 学习笔记 , Effective Java   |  

This chapter discusses several aspects of method design: how to treat parameters and return values, how to design method signatures, and how to document methods. Much of the material in this chapter applies to constructors as well as to methods. Like Chapter 5, this chapter focuses on usability, robustness, and flexibility.

Check parameter before using them for execution.

For example, Exception should be thrown if the parameter is invalid

总结，每次你写函数之前，应该做的第一件事就是边界检查（包括输入的参数是否合法）

Make defensive copy

这个类不是『不可改变』的本质原因是因为java constructor创建新对象时，传递参数是对象的内存地址。如果不特别在构造函数中把输入参数的值复制一遍，就可能造成意料不到的后果。同时，在做getter的时候，也要返回类中对象的拷贝而不是对象本身。

Design method signatures carefully

  1. choose a good name; It should follow the naming convention along with others in the same package.
  2. Dont provide too many methods or just make method name short for own convenience.
  3. Avoid long parameter list (aim 4 or fewer)
  4. For parameter types, faovr interface over class (use Map instead of HashMap)
  5. Prefer two-element enums over boolean

Use overloading judicously

Overload: 函数名字相同，输入参数类型不同。在编译时就已经选择好了
override: 子类和父类函数名字相同，输入参数相同，但在运行时动态决定调用哪个函数

safe: never use two overload methods with the same number of parameters.

Use varages judicously

Don’t retrofit every method that has a final array parameter; use varargs only when a call really operates on a variable-length sequence of values.

Return Empty String or Collections, not null

there is no reason ever to return null from an array- or collection-valued method instead of returning an empty array or collection

Write docs for all exposed API methods

@param
@return
@throw
@code

Posted on 2017-02-09   |   In 学习笔记 , Effective Java   |  

This chapter concerns creating and destroying objects: when and how to create them, when and how to avoid creating them, how to ensure they are destroyed in a timely manner, and how to manage any cleanup actions that must precede their destruction.

TBC

Consider static factories instead of constructors

instead of a constructor, use a static method that returns an instance of the class.

Pros:

  1. Static factory methods, unlike constructors, have names. And it will be easier for clients who use them.
  2. Static factory methods, unlike constructors, are not required to create a new object each time they are invoked. This allows immutable classes to use a preconstructed instance, or cache instances. The ability to return same objects from repeated invocations are called instance-controlled.
  3. Unlike constructors, static factory can return an object of any subtype of their return type, which gives you the flexibility in choosing the class of the returned object.
  4. Reduce the verbosity of creating parameterized type instances

Cons:

  1. Classes with only static factory methos but no public/proteceted constructors cannnot be subclassed.
  2. Static factory methods are not readily distinguished like constructors in API docs, which makes it hard to figure our how to instantiate a class.
